# Definition of Ready (DoR) — Banco Digital UdeA

> Una Historia de Usuario está **READY** para entrar al Sprint cuando cumple:

- [ ] Tiene título, descripción y criterios de aceptación claros (formato Gherkin para QA)
- [ ] Estimada en Story Points por el equipo
- [ ] Dependencias identificadas y desbloqueadas
- [ ] El modelo de datos que requiere está definido (por Estefanía)
- [ ] Los endpoints necesarios están identificados con su verbo HTTP y path
- [ ] Los roles que tienen acceso al endpoint están definidos
- [ ] El equipo de Calidad conoce los escenarios de prueba esperados
- [ ] Cabe en el Sprint sin superar la capacidad del equipo
